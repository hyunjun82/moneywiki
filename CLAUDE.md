## 🎯 독자 니즈 중심 글쓰기

### 핵심 질문
**"이 키워드 검색한 사람은 뭘 궁금해하나?"**

### 독자 정의
- **대상**: 20~80세 (지식 0 가정)
- **상황**: 해당 키워드 완전히 모름
- **목적**: 궁금증 100% 해결
- 못 알아들으면 → **즉시 이탈** (광고수익 0)

### 기승전결 구조
- **기**: 독자 상황 공감 + 왜 중요한지
- **승**: 핵심 개념 쉽게 + 용어 풀이
- **전**: 구체적 방법/절차/예시
- **결**: 정리 + 다음 행동 안내

### 글쓰기 핵심 기법

**1. 독자 상황으로 시작**
```
❌ "계약갱신청구권은 임차인이 갱신을 청구할 수 있는 권리입니다."
✅ "전세 2년 살았는데 계약 끝나간다고 연락왔어요. 2년 더 살 수 없나?
   있어요. 계약갱신청구권이라는 게."
```

**2. 일상 비유**
```
❌ "1회에 한해 2년 연장 청구 가능"
✅ "레스토랑 예약 연장권이라고 생각하면 돼요. 1회에 한해 2시간 더 쓸 수 있는 권리."
```

**3. 질문하고 바로 답하기**
```
❌ "내용증명을 통해 의사를 전달하는 것이 좋습니다."
✅ "카톡으로 보내도 되나요? 법적으로는 유효해요. 근데 나중에 '못 받았는데?'
   하면 증거 없어요. 그래서 내용증명이 안전해요."
```

**4. 짧은 문장, 한 가지만**
```
❌ "계약갱신청구권은 임차인이 임대인에게 갱신을 청구할 수 있는 권리로,
   1회에 한해 2년간 연장이 가능하며..."
✅ "1회만 쓸 수 있어요. 2년 더 살 수 있어요. 집주인은 특별한 사유 없이 거절 못 해요."
```

**5. 테이블 대신 텍스트**
```
❌ | 거절 사유 | 설명 |
   | 실거주 | 임대인 실거주 |

✅ **집주인이 직접 들어와서 살 때**
   "내가 그 집에서 살 거야" 하면 거절 가능해요. 본인 아니어도
   직계가족(부모, 자녀)이 살겠다고 해도 돼요.
```

### 벤치마크 사이트
- **토스**: 쉬운 단어, 짧은 문장, 깔끔함
- **뱅크샐러드**: 군더더기 없음
- **네이버 머니스토리**: 구체적 숫자, 맥락
- **뉴닉**: 구어체 (~거든요, ~인데요)
- **공통점**: 이모지 금지, 테이블 최소화, 텍스트 중심

## 📝 문체 & 구조 규칙

### 구어체 필수 패턴
- ~이에요, ~해요, ~하죠, ~거든요, ~인데요
- "쉽게 말해", "다시 말해", "예를 들면"
- "~할 수 있어요", "~해야 해요", "~하면 돼요"

### 단답형 금지
```
❌ "돼요." "있어요." "없어요."
✅ "네, 가능해요." "당연히 있어요." "아쉽지만 없어요."
```

### 테이블 사용 기준
**써도 되는 경우** (2가지만):
1. 신청 방법/절차 (단계별)
2. 비교표 (A vs B)

**쓰면 안 되는 경우**:
- 개념 설명 → 텍스트로
- 조건/요건 → 텍스트 후 필요시만
- 단순 나열 → 불릿 포인트

### 좋은 글 구조
```markdown
## 섹션 제목

[4-8문장 충분한 설명]
[전문용어 → 바로 풀어서]
[내부링크 자연스럽게]
[실제 예시/상황]

(필요시에만 간단한 테이블)
```

## 🔗 링크 전략 (SEO + 수익화)

### 1. 내부링크 - 위키 문서 간
- 형식: `[키워드](/w/슬러그)`
- 새창: ❌ (같은 탭에서 이동)
- 3개 이상 포함 필수
- 예: `[실업급여](/w/실업급여-수급조건)`, `[전입신고](/w/전입신고-방법)`

### 2. 내부 도구 링크 - 계산기 등
- 형식: `[계산기명](/calculators/슬러그)`
- 새창: ❌ (같은 탭에서 이동)
- **외부 계산기보다 내 사이트 계산기 우선 링크**
- 예: `[실업급여 계산기](/calculators/unemployment)`, `[양도소득세 계산기](/calculators/capital-gains-tax)`

### 3. 외부링크 - 정부기관, 신청/다운로드
- 형식: `[기관명](https://URL)` (Next.js가 자동으로 `target="_blank"` 처리)
- 새창: ✅ (새 탭 열림 + **전면광고 가능**)
- 행동 유도만 (신청, 조회, 다운로드, 확인)
- 예: `[국세청 홈택스](https://hometax.go.kr)`, `[정부24 신청하기](https://www.gov.kr)`

### 출처 표기 (이중 표기)
**본문 인라인**:
```markdown
[주택임대차보호법](https://www.law.go.kr/법령/주택임대차보호법)에 따르면...
```

**하단 섹션**:
```markdown
## 출처
```

### 세금/세율 (암기!)
| 구분 | 세율 | 조건 |
|------|------|------|
| 세액공제 | **16.5%** | 총급여 5,500만원 이하 |
| 세액공제 | **13.2%** | 총급여 5,500만원 초과 |
| 지연이자 | **연 20%** | 퇴직금 14일 초과 |

**금지 세율**: 15%, 12% (구버전 - 틀림!)

### 법정 기한
- 퇴직금 지급: **14일**
- 청구권 소멸: **3년**
- IRP 이전: **14일**

### 계산 공식
```
퇴직금 = 평균임금 × 30일 × (재직일수 ÷ 365)
지연이자 = 퇴직금 × 20% × (지연일수 ÷ 365)
세액공제 = 납입액 × 16.5% (또는 13.2%)
```

### 계산 예시 검증
**퇴직금 1,000만원, 30일 지연**:
```
10,000,000 × 0.20 × (30 ÷ 365) = 164,400원 ✅
```

## 📊 Google E-E-A-T (핵심만)

### 신뢰성(Trust) = 가장 중요
- **Experience**: 구체적 사례, 실제 상황
- **Expertise**: 정확한 정보, 법령 근거
- **Authoritativeness**: 정부기관, 법령 출처
- **Trustworthiness**: 위 3가지 결합

### 최종 질문 3가지
1. **독자가 다시 검색해야 하나?** → NO여야 함
2. **1페이지 다른 글보다 나은가?** → YES여야 함
3. **친구에게 공유하겠는가?** → YES여야 함
