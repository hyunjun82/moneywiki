## SEO 전략: 롱테일 키워드 ONLY

### 핵심 원칙 (절대 규칙)
1. **신규 도메인 = 단일 키워드 노출 불가능**
2. **모든 문서는 롱테일 키워드**
3. **단일 키워드 허브 문서 = 만들지 마라**

### 왜?
- 나무위키: 도메인 점수 높음 → "퇴직금", "연말정산" 노출됨
- 신규 도메인: 도메인 점수 없음 → 롱테일로만 노출 가능
- 200~300개 롱테일 문서로 조금씩 노출시키는 전략

### 올바른 예시
```
/w/연말정산-신용카드-공제-한도
title: "연말정산 신용카드 공제 한도"

/w/연말정산-세액공제-비과세
title: "연말정산 세액공제 비과세"

/w/연말정산-체크카드-공제
title: "연말정산 체크카드 공제"

/w/퇴직금-통상임금-계산
title: "퇴직금 통상임금 계산"

/w/퇴직금-중간정산-조건
title: "퇴직금 중간정산 조건"
```

### 금지 (절대 하지 마라)
```
/w/연말정산 → "연말정산" (단일 키워드 = 노출 안됨)
/w/퇴직금 → "퇴직금" (단일 키워드 = 노출 안됨)

title: "2026년 연말정산 총정리: ..." (블로그식 = 금지)
```

### 키워드 구조
```
[메인키워드] + [세부키워드] + [추가키워드]
연말정산 + 신용카드 + 공제 한도
퇴직금 + 통상임금 + 계산
실업급여 + 수급조건 + 자발적퇴사
```

## 문서 작성 규칙

### frontmatter 필수
```yaml
title: "롱테일 키워드"  # 검색어 그대로, 2~4단어
description: "..."
category: "세금"
keywords: [...]
lastUpdated: "YYYY-MM-DD"
datePublished: "YYYY-MM-DD"
summary: "3줄 요약"
sources:
  - name: "출처명"
    url: "https://..."
```

### 본문 출처 표기
- 출처는 본문 내 인라인으로 표기
- 예: `[국세청](https://nts.go.kr)에 따르면...`
