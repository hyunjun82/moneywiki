## SEO 전략: 롱테일 키워드 ONLY

### 핵심 원칙 (절대 규칙)
1. **신규 도메인 = 단일 키워드 노출 불가능**
2. **모든 문서는 롱테일 키워드**
3. **단일 키워드 허브 문서 = 만들지 마라**

### 왜?
- 나무위키: 도메인 점수 높음 → "퇴직금", "연말정산" 노출됨
- 신규 도메인: 도메인 점수 없음 → 롱테일로만 노출 가능
- 200~300개 롱테일 문서로 조금씩 노출시키는 전략

### 올바른 예시
```
/w/연말정산-신용카드-공제-한도
title: "연말정산 신용카드 공제 한도"

/w/퇴직금-통상임금-계산
title: "퇴직금 통상임금 계산"
```

### 금지 (절대 하지 마라)
```
/w/연말정산 → "연말정산" (단일 키워드 = 노출 안됨)
/w/퇴직금 → "퇴직금" (단일 키워드 = 노출 안됨)
title: "2026년 연말정산 총정리: ..." (블로그식 = 금지)
```

## 콘텐츠 작성 가이드라인 (가장 중요!)

### 0. 핵심 철학: 독자 니즈 중심 + 텍스트 위주 설명

**이 키워드를 검색하는 사람은 누구인가? 뭘 알고 싶어서 들어왔나?**

#### 독자 이해하기 (필수!)
- 검색하고 들어온 사람은 **완전 초보자**
- 이미 아는 사람은 검색 안 함 → **모르니까 검색함**
- 못 알아들으면? → **바로 이탈** (바운스율 폭등, 광고수익 0)
- 따라서: **처음 접하는 사람도 이해할 수 있게 설명**

#### 벤치마크 사이트 (4개 사이트 장점 통합)
- **토스**: 잡초제거, 쉬운 단어, 짧은 문장 → 깔끔하고 전문적
- **뱅크샐러드**: 깔끔한 UI, 정보 전달력 → 군더더기 없음
- **네이버 머니스토리**: 스토리텔링, 구체적 숫자 → 맥락 제공
- **뉴닉**: 구어체 어미 ("~거든요", "~인데요") → 친근함
- **공통점**: 이모지 안 씀, 테이블 최소화, 텍스트 중심

#### 절대 금지
- **이모지 사용 금지**: 🏠, 💰, 👇, ✅ 등 이모지 쓰지 마세요
- **블로그 말투 금지**: "알아봅니다", "정리해드릴게요" 등
- **과도한 강조 금지**: 모든 문장에 **굵은 글씨** 남발하지 마세요

#### 글쓰기 핵심 기법 (4개 사이트 통합)

**1. 독자 상황으로 시작하기**
```
❌ "계약갱신청구권은 임차인이 갱신을 청구할 수 있는 권리입니다."
✅ "전세 2년 살았는데 계약 끝나간다고 연락이 왔어요. 지금 집 마음에 드는데,
    그냥 2년만 더 살 수 없나? 있어요. 계약갱신청구권이라는 게."
```

**2. 일상 비유 필수**
```
❌ "계약갱신청구권은 1회에 한해 2년 연장을 청구할 수 있는 권리입니다."
✅ "레스토랑 예약 연장권이라고 생각하면 돼요. 좋은 자리에 앉았는데
    2시간 지나니까 나가래요. 근데 '저 1회에 한해 2시간 더 쓸 수 있는
    권리 있어요'라고 말하면? 식당은 특별한 이유 없이 거절 못 해요."
```

**3. 질문하고 바로 답하기**
```
❌ "내용증명을 통해 의사를 전달하는 것이 좋습니다."
✅ "카톡으로 보내도 되나요? 법적으로는 유효해요. 근데 문제는 나중이에요.
    집주인이 '그런 거 못 받았는데?' 하면요? 그래서 내용증명으로 보내는 게 안전해요."
```

**4. 짧은 문장, 한 가지만**
```
❌ "계약갱신청구권은 임차인이 임대인에게 갱신을 청구할 수 있는 권리로,
    1회에 한해 2년간 연장이 가능하며, 2020년 7월 주택임대차보호법 개정으로 도입되었습니다."
✅ "1회만 쓸 수 있어요. 2년 더 살 수 있어요. 집주인은 특별한 사유 없이 거절 못 해요."
```

**5. 테이블 대신 텍스트로**
```
❌ | 거절 사유 | 설명 |
   | 실거주 | 임대인 또는 직계가족 실거주 |

✅ **집주인이 직접 들어와서 살 때**
   "내가 그 집에서 살 거야" 하면 거절 가능해요. 본인 아니어도
   직계가족(부모, 자녀)이 살겠다고 해도 돼요.
```

#### 콘텐츠 핵심 원칙
1. **텍스트 중심**: 설명은 글로, 테이블은 최소한으로
2. **용어 설명 필수**: 전문용어 나오면 바로 쉽게 풀어서
3. **맥락 제공**: 왜 이게 중요한지, 언제 필요한지
4. **실제 상황 예시**: 추상적 설명 → 구체적 사례
5. **행동 유도**: 읽고 나서 뭘 해야 하는지 명확히

#### 나쁜 예시 (테이블 남발, 설명 없음)
```markdown
## 계약갱신청구권이란

| 항목 | 내용 |
|------|------|
| 정의 | 임차인이 갱신을 청구할 수 있는 권리 |
| 횟수 | 1회 |
| 기간 | 2년 |
```
→ "그래서 이게 뭔데? 나한테 왜 필요한데?" 답 없음

#### 좋은 예시 (텍스트로 충분히 설명)
```markdown
## 계약갱신청구권이란

계약갱신청구권은 쉽게 말해 "집주인, 나 2년 더 살게요"라고 말할 수 있는
법적 권리예요.

전세 계약이 끝나가는데 이사 가기 싫다면? 이 권리를 쓰면 돼요.
집주인이 특별한 사유 없이 거절하면 불법이에요.

2020년 7월부터 시행된 [임대차보호법](/w/임대차보호법)의 핵심 제도 중
하나죠. 최초 계약 2년 + 갱신 2년 = 총 4년까지 살 수 있어요.

1회만 쓸 수 있어서 타이밍이 중요해요. 언제, 어떻게 써야 하는지
아래에서 자세히 알려드릴게요.
```
→ 이게 뭔지, 왜 필요한지, 언제 쓰는지 다 설명됨

### 1. 문체: 구어체 필수

**원칙**: 정보는 정확하게, 말투는 쉽게

#### 좋은 예시 (이렇게 써라)
```
대항력이란 쉽게 말해 '내 보증금을 지킬 수 있는 힘'이에요.
집주인이 바뀌거나 집이 경매로 넘어가도 보증금을 받을 수 있죠.
이 힘을 얻으려면 실제로 이사하고 전입신고를 해야 해요.
```

#### 나쁜 예시 (사용 금지)
```
대항력은 임차인이 제3자에게 임대차의 내용을 주장할 수 있는
법적 효력을 의미합니다. 주택임대차보호법 제3조에 따라...
```

#### 문체 패턴
- "~이에요", "~해요", "~하죠", "~거든요", "~인데요"
- "쉽게 말해", "다시 말해", "예를 들면"
- "~할 수 있어요", "~해야 해요", "~하면 돼요"

#### 단답형 응답 금지 (중요!)
질문에 대한 첫 응답으로 **단답형("돼요.", "있어요.", "안 돼요.")** 사용 금지

**나쁜 예시**:
```
"월세도 계약갱신청구권 쓸 수 있어요?"
→ "**돼요.**" (너무 딱딱하고 어색함)

"집주인 바뀌어도 갱신청구권 있나요?"
→ "**있어요.**" (문맥 없이 뚝 떨어짐)
```

**좋은 예시**:
```
"월세도 계약갱신청구권 쓸 수 있어요?"
→ "월세도 당연히 쓸 수 있어요." (자연스러운 문장)

"집주인 바뀌어도 갱신청구권 있나요?"
→ "네, 그대로 유지돼요." (긍정 + 설명)

"4년 후에도 갱신청구권 쓸 수 있어요?"
→ "아쉽지만 못 써요." (감정 + 답변)
```

**원칙**: 응답의 첫 문장은 완전한 문장으로, 맥락이 있게 작성

### 2. 글 구조: 텍스트 중심, 테이블은 최소한

#### 테이블 사용 기준 (중요!)

**테이블 써도 되는 경우** (2가지만):
1. **신청 방법/절차**: 단계별로 뭘 해야 하는지
2. **비교표**: A vs B 명확한 비교가 필요할 때

**테이블 쓰면 안 되는 경우**:
- 개념 설명 → 텍스트로
- 조건/요건 → 텍스트로 설명 후 필요시 표로 정리
- 단순 나열 → 불릿 포인트나 텍스트로

#### 좋은 글 구조
```markdown
## 섹션 제목

[4-8문장 충분한 설명 - 이게 뭔지, 왜 중요한지, 언제 필요한지]
[전문용어 나오면 바로 풀어서 설명]
[내부링크 자연스럽게 포함]
[실제 예시나 상황 설명]

(필요시에만 간단한 요약 테이블)
```

#### 나쁜 글 구조
```markdown
## 섹션 제목

짧은 한 문장 설명.

| 항목 | 내용 |
|------|------|
| 이것 | 저것 |
| 저것 | 이것 |
| 또다른것 | 뭔가 |
```
→ 테이블로 때우면 안 됨. 글로 설명해야 함.

### 3. 링크 구분 (자동 인식!)

**내부링크 vs 외부링크 자동 구분**:

| 구분 | 대상 | 형식 | 예시 |
|------|------|------|------|
| **내부링크** | 우리 사이트 문서 | `/w/슬러그` | `[확정일자](/w/확정일자)` |
| **외부링크** | 실제 외부 사이트 | `https://URL` | `[정부24](https://www.gov.kr)` |

#### 내부링크 (우리 사이트)
- 대상: 전입신고, 확정일자, 대항력, 우선변제권 등 **우리가 작성한 문서**
- 형식: `[키워드](/w/슬러그-이름)`
- 효과: 사이트 내부 점수 상승

```markdown
[전입신고](/w/전입신고-방법)를 해야 해요.
[확정일자](/w/확정일자)까지 받으면 [우선변제권](/w/우선변제권-요건)도 생기죠.
```

#### 외부링크 (실제 사이트로 이동)
- 대상: 정부24, 홈택스, 법제처 등 **실제 외부 서비스/기관**
- 형식: `[이름](https://실제URL)`
- 효과: 클릭 시 외부 사이트로 이동 (전면광고 수익)

```markdown
[정부24](https://www.gov.kr)에서 신청하세요.
[주택임대차보호법](https://www.law.go.kr/법령/주택임대차보호법)을 참고하세요.
[홈택스](https://www.hometax.go.kr)에서 조회 가능해요.
```

#### 자동 판단 기준
- **우리 문서 있으면** → 내부링크 `/w/...`
- **실제 서비스/기관이면** → 외부링크 `https://...`

### 6. 목차 구조 (중요!)

**헤딩에 숫자 넣지 마세요! 사이트 테마가 자동 번호 매김**

**소제목에 검색 니즈 반영 (SEO)**:
- ❌ "왜 해야 하나요?" → 맥락 없음, 검색에 안 걸림
- ✅ "전입신고 해야 하는 이유" → 롱테일 키워드 포함

```markdown
## 첫번째 주제
내용...

## 두번째 주제
내용...

## 세번째 주제

### 하위 주제 A
내용...

### 하위 주제 B
내용...

## 네번째 주제
내용...
```

# 대항력 발생 시점

대항력이란 쉽게 말해 '내 보증금을 지킬 수 있는 힘'이에요.
집주인이 바뀌거나 집이 경매로 넘어가도 보증금을 받을 수 있죠.

## 대항력이란

전세로 들어갔는데 갑자기 집주인이 바뀌었어요. 새 집주인이 "나가세요"라고 하면
어떡하죠? 여기서 대항력이 있느냐 없느냐가 중요해요.

대항력이 있으면 새 집주인한테 이렇게 말할 수 있어요. "저 여기 살고 있고,
보증금 받을 권리 있어요. 제가 먼저예요." 법적으로 인정받는 힘이죠.

반대로 대항력이 없으면? 새 집주인이 내보낼 수 있어요. 보증금도 날릴 수 있고요.
그래서 전세 계약하면 반드시 대항력부터 갖춰야 해요.

[주택임대차보호법](https://www.law.go.kr/법령/주택임대차보호법) 제3조에서
이 권리를 보장하고 있어요. 법이 세입자 편이라는 뜻이죠.

## 대항력 얻는 조건

대항력을 얻으려면 두 가지가 필요해요. 하나라도 빠지면 안 돼요.

**첫째, [전입신고](/w/전입신고-방법)**를 해야 해요. 주민센터 가서 "이 주소로
이사 왔어요"라고 신고하는 거예요. 요즘은 [정부24](https://www.gov.kr)에서
온라인으로도 돼요.

**둘째, 실제로 그 집에서 살아야 해요.** 법에서는 이걸 '점유'라고 불러요.
전입신고만 해놓고 실제로 안 살면 대항력 인정 안 돼요. 이사 짐 옮기고
실제로 거주해야 해요.

## 대항력 생기는 시점

여기가 핵심이에요. **대항력은 전입신고한 '다음 날 0시'에 생겨요.**
당일 바로 생기는 게 아니에요!

예를 들어볼게요. 3월 1일에 전입신고했다면, 대항력은 3월 2일 0시에 생겨요.
그런데 만약 3월 1일 오후에 집주인이 집을 팔았다면? 3월 1일에는 아직
대항력이 없으니까, 보증금을 보호받지 못할 수 있어요.

그래서 **잔금일 당일 바로 전입신고하는 게 중요해요.** 잔금 치르자마자
주민센터 가세요. 하루라도 빨리 대항력 확보해야 안전해요.

[확정일자](/w/확정일자)까지 같이 받으면 [우선변제권](/w/우선변제권-요건)도
생겨요. 경매 넘어갔을 때 보증금을 먼저 받을 수 있는 권리죠. 전입신고하러
가는 김에 확정일자도 같이 받아두세요. 600원이면 돼요.
