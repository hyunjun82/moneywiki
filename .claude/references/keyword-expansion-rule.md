# 키워드 확장 규칙

## 핵심 원칙

### 하나의 질문 = 하나의 문서
- 사용자가 여러 질문을 주면 **각 질문마다 별도 문서** 작성
- 비슷한 주제라도 질문이 다르면 **별도 확장**
- 각 질문을 4-5개 키워드로 점진적 확장

**예시:**
```
질문 1: "묵시적 갱신을 거절하려면 어떻게 해야 하나요?"
→ 문서 1개 (키워드 4-5개)

질문 2: "묵시적 갱신 후 3개월이 지나면 어떻게 되나요?"
→ 문서 1개 (키워드 4-5개)

질문 3: "상가 묵시적 갱신 후 계약 해지를 어떻게 하나요?"
→ 문서 1개 (키워드 4-5개)

총 3개 질문 = 총 3개 문서
```

---

### title vs keywords
- **title**: 문서 제목 (검색 노출용 SEO)
- **keywords**: 소제목 후보 (본문 ## 섹션 헤딩으로 사용)

### 키워드 요건
- 각 키워드는 **의미적으로 다른 측면**
- 소제목으로 쓸 수 있을 정도로 **구체적**
- 중복 의미 제거 필수
- **4-5개 고정**

---

## 중복 판별 기준

❌ **중복 (제외해야 함):**
- "확인" = "확인방법" (같은 의미)
- "신청" = "신청 방법" (같은 의미)
- "계산" = "계산법" (같은 의미)
- "조회" = "조회 방법" (같은 의미)

✅ **별개 (포함해야 함):**
- "신청" ≠ "조건" (행위 vs 요건)
- "비용" ≠ "선임" (금액 vs 행위)
- "계산" ≠ "납부" (계산 vs 납부 행위)
- "확인" ≠ "조건" (조회 vs 요건)

## 예시 1: 실업급여

```yaml
title: 실업급여 개별연장급여 신청 조건
keywords:
  - 실업급여
  - 실업급여 개별연장급여
  - 실업급여 개별연장급여 신청
  - 실업급여 개별연장급여 조건
```

**확장 분석:**
- 베이스: 실업급여
- 토픽: 개별연장급여
- 액션: 신청 (어떻게?)
- 조건: 조건 (누가? 언제?)

**소제목 예상:**
- ## 실업급여란
- ## 실업급여 개별연장급여, 정확히 뭐예요?
- ## 실업급여 개별연장급여 신청 방법
- ## 실업급여 개별연장급여 조건

## 예시 2: 양도소득세

```yaml
title: 1가구 2주택 양도소득세 면제 조건
keywords:
  - 양도소득세
  - 1가구 2주택 양도소득세
```

**확장 분석:**
- 베이스: 양도소득세
- 대상: 1가구 2주택
- 혜택: 면제 (얼마나 줄어?)
- 조건: 조건 (언제 면제?)

**소제목 예상:**
- ## 양도소득세란
- ## 1가구 2주택 양도소득세, 얼마나 내요?
- ## 1가구 2주택 양도세 면제 혜택
- ## 1가구 2주택 양도소득세 면제 조건

## 작업 흐름

### 사용자가 할 일
1. 질문 키워드 수백~수천 개 텍스트/이미지로 제공

### Claude가 할 일
1. `keyword-expansion-rule.md` 읽고 패턴 파악
2. 제공된 키워드들 중복 제거
3. 예시 패턴 참고해서 4-5개 키워드 정리
4. title + keywords 생성
5. 위기브 스타일로 글 작성

---

## 중복 체크 프로세스 (필수!)

### Step 1: 사용자 제공 키워드 중복 확인
사용자가 준 Google 연관검색어/관련질문 목록에서 중복 제거

### Step 2: 기존 wiki 파일 검색
```bash
# 각 키워드별로 검색
Grep("양도소득세", path="content/wiki", output_mode="files_with_matches")
Grep("1가구 2주택 양도소득세", path="content/wiki", output_mode="files_with_matches")
```

### Step 3: 중복 발견 시
- 해당 키워드 제외하고 다른 키워드로 대체
- 또는 기존 문서에 내용 추가 (신규 문서 X)

---

## 체크리스트 (글 작성 전)

- [ ] **사용자 제공 키워드 목록에서 중복 제거**
- [ ] **기존 wiki 파일에서 동일 키워드 Grep 검색**
- [ ] 4-5개 키워드가 서로 중복되지 않는지 확인
- [ ] 베이스 키워드가 카테고리에 맞는지 확인
- [ ] Title이 풀 롱테일인지 확인
- [ ] 각 키워드가 소제목으로 쓸 수 있을 정도로 구체적인지 확인

---

## 금지

- 키워드 6개 이상 ❌
- 키워드 3개 이하 ❌
- 베이스 키워드 없이 바로 롱테일 ❌
- 같은 의미 키워드 중복 ❌ (예: "확인" + "확인방법")
